# 毎朝 04:30 JST — 依頼の生成

cron: `30 19 * * *`（UTC）

デモ用に、ユーザー役として Slack に改善依頼を投げる。
30 分後に [`morning-loop`](morning-loop.md) がこれを読んで判断・実装する。

---

loop-engineering-lab のデモ用に、Slack へ改善依頼を 3 件投稿してください。

このリポジトリは「Slack の依頼 → Issue → 実装 PR」を無人で回す実証です。あなたの役割は **依頼を投げる側（非エンジニアのユーザー役）** です。

30 分後に別セッションのルーチンがこれを読んで、依頼かどうか判断し、Issue にして実装します。

## まず現状を把握する

1. README.md を読む
2. app/weather.py を読んで、今何を取得・返却しているか確認する
3. frontend/src/ を読んで、**画面に何が表示されているか**確認する。
   API にあるのに画面に出ていない項目は、画面表示の依頼の自然なネタになる
4. GitHub の Issue（state=all）を見て、**既に依頼済み・実装済みのものを避ける**
5. Slack でチャンネル C0BJK86F2K0 の過去投稿（直近 50 件）を読み、**同じ依頼を繰り返さない**

**未処理の依頼が 10 件以上溜まっていたら、今回は投稿しないで終了する。** 処理が追いついていないのに積み上げても意味がないため。

## 依頼のネタ（この範囲から選ぶ）

**A. Open-Meteo の未使用パラメータを追加する**

ドキュメント: https://open-meteo.com/en/docs
未使用の例: apparent_temperature（体感温度）、precipitation（降水量）、surface_pressure（気圧）、wind_direction_10m（風向・度数）、weather_code（天気コード）、cloud_cover（雲量）、uv_index（hourly）

**上記が使い尽くされてきたら、ドキュメントを見て自分で新しいパラメータを探すこと。**
ただし **無料で /v1/forecast から取れるものに限る**。有料プランや別 API（大気質、海洋など）は選ばない。

**B. レスポンス形式の改善**
並び順、欠測の扱い、メタ情報（取得時刻や地点名）、単位の見せ方 など

**C. 画面（デモページ）の小改修**

デモ: https://d10o14tv6y0g4t.cloudfront.net （frontend/ ディレクトリ。React + Vite、S3 + CloudFront 配信）

API では返しているのに画面に出ていない項目を表示する依頼、見た目や読みやすさの要望など。
例:「グラフに湿度も重ねてほしい」「日の出・日の入りの時刻を画面にも出して」
「今の気温の数字を大きく出して」「スマホだとグラフが見づらい」

ただし **1 PR に収まる規模に限る**。画面の作り直し、ページの追加、認証などは選ばない。

**選ばないもの**: 新規サブシステム（DB、認証、機械学習、フロントの作り直し）、インフラ変更

## 文体：毎回ランダムに混ぜる

**あなたはエンジニアではありません。** 整った仕様書を書かないでください。
以下の型から 3 件分を選び、**毎回違う組み合わせになるようランダムに混ぜて**ください。

1. **明確な機能追加** — 例:「風向きも返してほしい。度数だと分からないので北西とかにして」
2. **あいまい・ぼんやり** — 例:「天気の情報が少ない気がする。もうちょっとなんとかならない？」
3. **要望が混ざる** — 例:「降水量追加して。あとついでに気圧も」
4. **不具合の報告（体裁）** — 例:「湿度がたまに空で返ってくることがある気がする」
5. **依頼ではない雑談** — 例:「今日暑いですね。体感 35 度とかでした」
6. **画面への感想・要望** — 例:「グラフ、気温だけだと寂しいね。雨が降るかも見たい」

**3 件のうち、少なくとも 1 件は「あいまい」か「雑談」にしてください。**
ネタ元は A・B・C に偏りが出ないよう散らしてください。特に最近は A（API 追加）に
偏っているので、**当面は 3 件のうち 1 件以上を C（画面）にしてください。**
読み取り側が「弾く判断」「質問を返す判断」をできるか検証したいためです。全部明確な依頼だと検証になりません。

## 投稿の仕方

- Slack のツールでチャンネル C0BJK86F2K0 に **1 件ずつ別のメッセージとして** 投稿する
- 1 件あたり 1〜3 行程度。長々と書かない
- 絵文字のリアクションは不要（読み取り側は内容で判断する）
- 「これはテストです」などのメタ発言は入れない。実際のユーザーのように書く

## 制約

- **Issue も PR も作らない。** あなたは依頼を投げるだけ
- **コードを変更しない。** 読むだけ
- 既存の依頼と重複しない

## 報告

最後に、投稿した 3 件の本文と、それぞれどの型（1〜5）を意図したかを報告してください。
